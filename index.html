ตาราง “bookings”
รหัส / Job No. ผู้ขอ / แผนก รถ ปลายทาง ช่วงเวลา สถานะ
สถาปัตยกรรม: แอปนี้คุยกับ Supabase อย่างไร

แอปนี้เป็น Static SPA วางบน GitHub Pages และเชื่อมกับฐานข้อมูล PostgreSQL บน Supabase ผ่าน JS Client โหลดข้อมูลทั้งหมดทีเดียวตอนเปิดแอป:

const { data } = await sb
  .from('bookings')
  .select('*')
  .order('created_at', { ascending:false });

การตรวจเวลาซ้อนทับมี 2 ชั้น: (1) Client query ตรวจก่อนบันทึก (2) Postgres EXCLUDE constraint กันการจองทับซ้อนระดับฐานข้อมูล แม้มีผู้ใช้พร้อมกันก็ไม่ชนกัน

แบบฟอร์มขอใช้ยานพาหนะ

กรอกรายละเอียดให้ครบถ้วน ระบบจะตรวจสอบเวลาซ้อนทับให้อัตโนมัติ

กรอกเป็นชื่อสถานที่ หรือพิกัด lat,lng ก็ได้

การตรวจเวลาซ้อนทับ

ก่อนบันทึก ระบบจะตรวจทุกการจองของรถคันเดียวกันที่สถานะ Approved หรือ Pending หากช่วงเวลาทับซ้อน คำขอจะถูกปฏิเสธทันที

(newStart < existEnd) && (newEnd > existStart)
ขั้นตอนตอนกดส่งคำขอ

กด Submit จะ e.preventDefault() เพื่อไม่ให้หน้ารีเฟรช จากนั้น:

(1) ตรวจเวลาซ้อนทับ → (2) insert ลงตาราง bookings พร้อมเจน BK-PSP102-{timestamp} สถานะเริ่มต้น Pending → (3) แจ้งอีเมลผู้อนุมัติผ่าน Supabase Edge Function → (4) รีเฟรชเฉพาะตาราง+ปฏิทิน

การอนุมัติคำขอ

ตรวจสอบสถานะและดำเนินการอนุมัติ/ปฏิเสธคำขอใช้ยานพาหนะ

ผู้มีสิทธิ์อนุมัติ

จัดการรายชื่อผู้มีสิทธิ์อนุมัติคำขอใช้รถ และอีเมลสำหรับแจ้งเตือน

ผู้มีสิทธิ์ขอใช้ยานพาหนะ

จัดการรหัสเข้าใช้งาน (PSP-xxxxxx) และชื่อ-สกุลของผู้มีสิทธิ์ยื่นคำขอ

จัดการยานพาหนะ

เพิ่ม / แก้ไข / ลบ ข้อมูลรถ และปรับสถานะพร้อมใช้งาน
